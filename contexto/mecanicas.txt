MECÁNICAS DEL JUEGO WILD
=========================

🎮 MOVIMIENTO DEL JUGADOR
--------------------------

CONTROLES
---------
- WASD: Movimiento en plano XZ (adelante/atrás, izquierda/derecha)
- Mouse: Rotación de cámara FPS (horizontal y vertical)
- ESC: Abrir/cerrar menú de pausa
- Shift: (Reservado para correr en futuro)

FÍSICA
------
- CharacterBody3D: Cuerpo del jugador con gravedad
- CollisionShape3D: Colisión esférica para el jugador
- StaticBody3D: Suelo estático con colisión de caja
- BoxShape3D: Colisiones para árboles (1x4x1m)

CÁMARA
------
- Cámara hija del jugador (se mueve con él)
- Rotación libre en 3D con mouse
- Límites de rotación: -90° a +90° (pitch)
- Captura de cursor al iniciar juego

🌍 MUNDO PROCEDURAL
--------------------

GENERACIÓN
---------
- Terreno procedural con ruido Perlin y alturas variables (-100 a 1000 metros)
- Sistema de chunks de 100x100 metros para optimización
- Chunk inicial dinámico basado en posición del jugador
- Spawning procedural de árboles con colisiones
- Textura de hierba seamless con UV escalados

SISTEMA DE COLISIONES MODULAR
----------------------------
- CollisionHandler: Sistema modular para gestión de colisiones del jugador (164 líneas)
- Initialize(): Configuración inicial con Player, PlayerController y NetworkManager
- SetupCollisionDetection(): Configuración de Area3D para detección de barreras
- Sistema de capas: Jugador en capa 1, barreras en capa 2
- Area3D con CapsuleShape3D: Detector de colisiones más grande que el jugador
- OnBodyEntered()/OnBodyExited(): Manejo de eventos de colisión
- HandleBarrierCollision(): Teletransporte automático al detectar barreras de chunk
- Integración con NetworkManager para sincronización de colisiones
- Teletransporte automático al detectar límites de chunk

MOVIMIENTO MODULAR
------------------
- PlayerController: Sistema modular para movimiento WASD y cámara (250 líneas)
- Rotación de cámara con mouse y límites configurables (-89° a +89°)
- Sistema de eventos: PlayerMoved y CameraRotated para comunicación
- Modo red: _isNetworkMode para respuesta inmediata en multijugador
- Congelación/Descongelación: Freeze(), Unfreeze(), FreezePlayer(), UnfreezePlayer()
- Posición: GetPlayerPosition(), SetPlayerPosition(), GetPlayerGlobalPosition(), SetPlayerGlobalPosition()
- Cámara: GetCameraAngles(), SetCameraAngles(), ApplyCameraRotation()
- Terreno: AdjustToTerrain(), SetPlayerHeight() para adaptación al terreno
- Integración completa con Logger y NetworkManager
- Velocidad configurable: 5 m/s (caminar normal) con MouseSensitivity ajustable

PERSISTENCIA MODULAR
--------------------
- PlayerPersistence: Sistema modular para gestión de persistencia (399 líneas)
- SavePlayerData(): Guardado de posición y rotación del jugador en JSON
- LoadPlayerData(): Carga robusta con validación de datos y fallback
- HasSavedData(): Verificación eficiente de existencia de datos
- GetWorldPath() y GetPlayerId(): Lógica de rutas e IDs encapsulada
- Auto-save: Sistema automático cada 5 segundos (SaveIntervalMs = 5000)
- Métodos de conveniencia: LoadPlayerPosition(), SaveOnExit() para GameWorld
- Validación completa: IDs de jugador, posición global vs local, timestamp
- Manejo de errores robusto con try-catch y logging detallado
- JSON serializado con JsonSerializerOptions y CamelCase
- Integración con CharacterManager para IDs consistentes
