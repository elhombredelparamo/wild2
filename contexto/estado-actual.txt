🏗️ ARQUITECTURA DEL SISTEMA
----------------------------

SISTEMA DE PERSONAJES
- CharacterManager: Singleton global para gestión de personajes
- IDs únicas de 16 caracteres basadas en MD5 con sal (ej: "a1b2c3d4e5f6g7h8")
- Perfiles guardados en user://characters/profiles/
- Personaje por defecto "jugador" creado automáticamente
- IDs consistentes entre guardado y carga
- Soporte multi-mundo y multi-jugador
- Sistema de IDs: MD5(nombre + timestamp + random) → primeros 16 chars
- Menús completos: CharacterSelectMenu, CharacterCreateMenu
- Eliminación inteligente: siempre queda al menos 1 personaje
- "jugador" eliminable si hay otros personajes disponibles

SISTEMA DE MUNDOS
- WorldManager: Gestión de mundos independientes
- Cada mundo en directorio propio: user://worlds/{nombre}/
- Datos de jugadores por mundo: user://worlds/{nombre}/player/{playerId}.json
- Sistema de guardado automático cada 5 segundos

SISTEMA DE RED
- GameServer: Servidor TCP para multi-jugador
- GameClient: Cliente TCP con sincronización
- IDs únicos: player_1, player_2, etc.
- Posición y rotación sincronizadas

SISTEMA DE UI
- MainMenu: Menú principal con imagen de fondo
- NewGameMenu: Creación de nuevos mundos
- WorldSelectMenu: Selección y gestión de mundos
- PauseMenu: Menú de pausa durante el juego
- Navegación fluida entre menús sin cerrar ventanas

🏗️ ARQUITECTURA DEL SISTEMA DE BARRERAS
---------------------------------------------
✅ **Barreras físicas implementadas** - StaticBody3D con 4 paredes invisibles
✅ **Altura completa** - 1200m para cubrir rango (-100 a 1000m)
✅ **Grosor optimizado** - 1m efectivo pero minimalista
✅ **Capas correctas** - Barreras en capa 2, jugador con CollisionMask = 2
✅ **Eliminación dinámica** - Barreras eliminadas al cargar chunks adyacentes

🎯 DETECCIÓN Y TELETRANSPORTE
------------------------------------
✅ **Detección centralizada** - GameWorld.cs detecta colisiones con barreras
✅ **Teletransporte automático** - 2 metros hacia atrás al tocar barrera
✅ **Notificación al servidor** - Mensajes BLOCKED_POS para sincronización
✅ **Dirección inteligente** - Hacia centro del chunk si jugador está quieto
✅ **Logging completo** - Registro de todas las interacciones con barreras

## 📋 Sistema de Backups
- **Control total:** Solo tú decides cuándo hacer backup
- **Comandos disponibles:**
  - `powershell -ExecutionPolicy Bypass -File backup.ps1 create "motivo"` - Crear backup
  - `powershell -ExecutionPolicy Bypass -File backup.ps1 restore` - Restaurar último backup
- **Sin automatización:** No más backups automáticos
- **Sin conflictos:** Formato ZIP evita duplicados de Godot
